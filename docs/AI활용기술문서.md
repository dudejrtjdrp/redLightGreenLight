# AI 활용 기술 문서

무궁화 밸런스 게임 개발 과정에서의 AI(Claude) 활용 내역 정리. 프로젝트 진행에 따라 계속 최신화한다.

## 활용 개요

| 구분 | 내용 |
|---|---|
| 사용 도구 | Claude (Cowork 모드) — 코드 작성/리팩토링, 헤드리스 시뮬레이션, 문서화 |
| 활용 방식 | 자연어 요구사항 → 코드베이스 분석 → 구현 → 시뮬레이션 검증 → 문서 갱신 |

## 세션 로그

### 2026-07-16 — Phase 9: 홈 화면 + 봇 모드 + 시뮬 기반 점수 밸런스

**요구 프롬프트(요약)**: "홈 화면 추가, 봇 모드/디버그 모드 분리, 길 확장, 봇 인원수 지정(최대 6),
상단 1분 30초 타이머, 술래/플레이어 승패 결과, 시뮬레이션으로 점수제 최적 밸런스 탐색 후 적용."

**AI가 수행한 작업**:

1. **홈/결과 화면** (`src/ui/HomeScreen.ts`, `src/ui/ResultScreen.ts` 신규)
   - 모드 선택(봇/디버그) + 봇 인원 스테퍼(1~6) + 조작 안내.
   - 종료 시 술래 vs 1등 무버 비교로 승패 판정, 통합 순위표, 다시 하기(sessionStorage로 설정 유지)/홈으로.
2. **봇 AI** (`src/gameplay/BotController.ts`, `src/config/BotConfig.ts` 신규)
   - `MoverInput` 인터페이스 신설(`Mover.ts`) — 사람 입력(InputController)과 봇이 동일 규칙(PlayerSystem)을 사용, 봇 전용 치트 없음.
   - 개성(공격성/반응속도/출발지연) 샘플링, FAST "치고-멈추기" 버스트 + 그린 장기화 시 예측 정지,
     RED 딜레마(경고 소모 보정 vs 짐 흘림) 확률 결정, 좌우 배회.
3. **멀티 무버 렌더** (`GameRenderer.ts` 개정, `CharacterView.create`에 모델 경로 파라미터)
   - 무버별 아바타 Map, 봇은 KayKit 모델(Mage/Ranger/Rogue/Rogue_Hooded) 순환 배정.
   - 화면 흔들림은 플레이어 자신 이벤트에만 적용(멀미 방지).
4. **길 확장**: 레인 반폭 4.5→7.0(`GameBalance.track`), 도로 스케일 1.6→2.5(`MapConfig`),
   카메라 상향(`EffectConfig.camera`), 바닥 46폭. 회수 판정에 x축 거리 추가(`CargoSystem` 멀티 대응 버그픽스).
5. **타이머/종료**: 라운드 90s 고정, `RoundStateMachine.remaining` 게터, 상단 중앙 카운트다운(`UIOverlay`).
6. **점수 밸런스 시뮬레이션** (`scripts/balanceSim.ts`, `scripts/verifyFinal.ts` 신규)
   - 실제 게임 코드를 헤드리스로 4,000라운드 구동해 이벤트 분포 수집 → 노브 그리드 해석 평가.
   - **구조적 발견**: 술래 점수원은 인원수 비례, 1등 무버 점수는 인원 무관 → 단일 배율로 인원별 밸런스 불가.
   - **채택**: K 28→20, dropPointScale 0.6→1.8, catchBonusScale 1→0.9, **인원수 보정 ×(4/무버수) 신규**.
   - **검증**(실제 정산 코드, 600라운드×5구간): 술래 승률 43~58% (보정 전 14~92%).

**AI 활용 기법 메모**:
- 밸런스 탐색 시 "이벤트 분포 수집(1회)과 점수 노브 평가(그리드)를 분리"해 시뮬 비용을 수십 배 절감.
- 봇 파라미터(예측 정지 등)는 시뮬 통계(탈락률/완주율)를 보며 반복 튜닝.

**팀원 기여**: 이 세션의 설계 방향 결정·요구사항 정의 — 이성효. (김태우 담당분은 팀원 롤 기술서 참고)

### 2026-07-17 — Phase 10: 줄서기 연출 + 조작감 + 인원수 공평 재밸런스

**요구 프롬프트(요약)**: "탈락자가 술래 왼쪽으로 걸어가 줄 서게, 인원수와 무관하게 공평한
점수 밸런스, 전진 중 좌우 균형 조작이 방향과 무관하게 쏠리는 문제 개선."

**AI가 수행한 작업**:

1. **탈락자 줄서기** (`src/gameplay/CaughtMarchSystem.ts` 신규)
   - `mover:caught` 구독 → 잡힌 순서대로 술래 왼쪽(-x)에 자리 배정, 걸어가 정렬.
   - `mover.velocity`를 채워 렌더 레이어(걷기 애니/조향)가 자연히 따라오게 함.
     탈락자도 걷기 애니가 나오도록 MoverView/CharacterView의 caught 게이트 제거.
   - 술래 위치 상수(`EffectConfig.seeker.standoffZ`)를 SeekerView와 공유(하드코딩 제거).
2. **균형 조작감 개선** (원인 진단 → `BalanceConfig.counterLeanCouple` 신규)
   - 원인: 쏠림(drift)은 속도·종반 배율로 최대 ~5배 커지는데 되잡기 torque는 고정
     → 고속·종반에 입력이 drift에 압도되어 "누른 방향과 무관하게 쏠리는" 느낌.
   - 해결: 되잡기 조작력을 같은 배율에 연동(couple 0.8) + torque 10→12.
   - 부작용 대응: 봇이 완벽 보정해 낙하 경제가 죽는 문제(낙하 0.1/라운드) →
     봇에 보정 반응 지연(0.15~0.4s)과 불완전 입력(세기 0.6) 부여로 복구(2~7.6/라운드).
3. **인원수 공평 재밸런스** (시뮬 4,000라운드 × 노브 그리드 재탐색)
   - 발견: 가속형 잡기 보너스 `c(c+1)/2`가 다인전 편차의 주범 → **선형(1명당 고정점)** 옵션
     추가(`catchBonusMode`) 후 그리드 탐색.
   - 채택: K=10, dropScale=1.0(낙하 1개=1점 대칭), 잡기 선형 1.8/명, 인원수 보정 ×(4/무버수).
   - 검증(실제 정산 코드 600라운드×5구간): 술래 승률 41.8~53.0%, 평균 점수 술래 8.7~9.0 vs
     1등 무버 7.8~8.9 — 전 구간 수렴.

**AI 활용 기법 메모**: 조작 파라미터 변경이 봇 행동 → 이벤트 분포 → 점수 밸런스로 연쇄되므로,
"조작 수정 → 봇 인간화 보정 → 분포 재수집 → 노브 재탐색" 순서로 파이프라인을 재실행함.

### 2026-07-17 — Phase 11: 탈락자 짐 흘리기 + 타임업 탈락 + 낙하 짐 시각 수정

**요구 프롬프트(요약)**: "잡히면 가는 길에 짐을 다 떨구게(뒤 사람이 줍게), 90초 타임업 시
그대로 탈락 처리 후 정산, 맵에 떨어진 짐이 안 보이는 문제, 낙하물이 짐과 다른 작은 정사각형인 문제."

**AI가 수행한 작업**:

1. **탈락자 짐 흘리기** (`CaughtMarchSystem` 확장): 줄서기 경로에 짐을 균등 분배해 낙하
   (경로/(짐+1) 간격, 0.8~2.4 클램프). 도착 시 잔여분 일괄 낙하. 라운드 진행 중 낙하는
   기존 규칙대로 술래 가점, 라운드 종료 후엔 무가점 forfeit(`CargoSystem.dropFrom(awardSeeker)`).
2. **타임업 탈락** (`ScoreSystem.settle`): 정산 직전 ALIVE(미완주) 무버를 전원 탈락 처리
   (`emitMoverCaught` → 술래 잡기 집계 + 줄서기 연출 자동 연동). 생존 보너스는 완주자만.
3. **낙하 짐 안 보임 원인 수정** (`MapView`): 도로 스케일 2.5 확대로 도로 "상판"이 y>0으로
   올라와 낙하 짐·완주선이 묻혀 있었음 → 타일 bbox 실측으로 상판을 y=0에 정렬.
4. **낙하 짐 시각 통일** (`EffectSystem`/`DroppedItemsView`): 낙하 비행체·트랙 잔류 짐을
   들고 있던 짐 박스와 동일한 크기(0.56×0.28×0.56)·색(0xffcc44)으로 통일, 밑면이 도로에 닿게 배치.
5. **재밸런스**: 새 규칙(잡히면 짐 수확)으로 술래 점수원이 급증 → 재시뮬 그리드 탐색.
   채택: K=20, drop=1.0, 잡기 선형 0.8/명(짐 수확이 잡기 가치를 대체하므로 하향),
   인원수 보정 power 1→1.3. 검증(실제 정산 코드 600라운드×5구간): **전 구간 술래 승률 46.7~52.2%**.
